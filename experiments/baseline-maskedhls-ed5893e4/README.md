# baseline-maskedhls

Establishes the "Standard MaskedHLS (Sarma et al., 2024), register-only
compiler" baseline: latency (cycles), area (gates), power (uW, estimated),
and TVLA t-value for two DOM-AND masked cryptographic Sbox gadgets (AES
SubBytes / Canright decomposition, and the PRESENT Sbox), both taken as
real, already-generated RTL outputs from the upstream open-source MaskedHLS
project (https://github.com/nilotpolas/MaskedHLS), specifically the
register-balanced ("retimed") netlists under `4. SecureHLS-main/test/*/DOM/`
in that repo — the actual final product of MaskedHLS's register-at-gadget-
boundary glitch-stopping strategy described in the experiment's hypothesis.

## Why this pipeline looks the way it does

This pod has **no root** (no `apt-get install verilator/iverilog/gcc`) and
**no GPU**. The only EDA tooling available is `yosys` via the pip-installable
`yowasp-yosys` WebAssembly build. There is therefore no C/C++ compiler
anywhere on the pod, which rules out Verilator (needs a C++ compiler to
build its generated model) and Icarus Verilog (not available as a wheel).

So instead of "synthesize with yosys, simulate with verilator," this
pipeline is: **synthesize the RTL to a generic (technology-independent)
gate-level netlist with yosys, then simulate that exact JSON netlist
directly in Python/numpy**, vectorizing the whole batch of Monte-Carlo TVLA
traces across the numpy array dimension so millions of traces run in
minutes instead of a per-trace Python loop. `code/sim/netlist_sim.py` is a
from-scratch bit-level logic simulator for the small set of 2-input gate
primitives (`$_AND_`, `$_OR_`, `$_XOR_`, `$_NOT_`, `$_MUX_`, `$_DFF_P_`)
that `techmap; simplemap` (no ABC — ABC hung on this HLS-generated
netlist's ~150k pre-optimization RTL cells inside the WASM sandbox) emits.
This makes "compiled RTL" and "the thing that got simulated" the exact same
artifact, and the power/TVLA analysis is genuinely simulation-derived, not
hand-waved.

Correctness of that homemade simulator is not assumed — it is checked twice:
against the original MaskedHLS authors' own reference oracle for AES
(`output_original.txt`, generated by their own iverilog run) and against a
bit-exact Python transliteration of the original `present_domand.c` source
for PRESENT. Both checks are re-run automatically at the start of every
`run_benchmark.py` invocation (see `functional_check_aes`/
`functional_check_present`) — this doubles as the "functional correctness /
formal equivalence check" the automated methodology reviewer flagged as
missing.

## Hardware requirements

CPU-only. No GPU. ~2 GB RAM. Runs in plain CPython + numpy (no CUDA, no
special hardware). Synthesizing the AES netlist takes ~25s; loading it into
the Python simulator takes ~8s; the full 10M-trace TVLA sweep for AES takes
roughly 60–70 minutes single-threaded (PRESENT's 10M-trace sweep takes
~15 seconds — it is a much smaller circuit).

## Install

```bash
pip install -r requirements.txt
```

## Run

```bash
cd code
python3 run_benchmark.py present 10000000 50000   # ~15s
python3 run_benchmark.py aes     10000000 20000   # ~60-70 min
python3 aggregate_results.py                       # writes ../results.json, ../report.md figures
```

`run_benchmark.py <benchmark> <n_traces_per_class> <batch_size>`:
1. Synthesizes `rtl_src/<benchmark>_sbox_dom_reg.v` with yosys if
   `../compiled_rtl/<benchmark>_sbox_gatelevel.json` doesn't already exist.
2. Loads the gate-level JSON netlist into `sim/netlist_sim.Netlist`.
3. Determines pipeline latency (cycles) from the DFF structure.
4. Runs the functional-equivalence check against the golden reference.
5. Runs fixed-vs-random TVLA (`sim/power_tvla.run_tvla`) for `n_traces`
   traces per class, streaming Welch's t-test accumulators so memory stays
   flat regardless of trace count. Prints progress every batch.
6. Writes `results_<benchmark>.json`.

No random seed sweep is used (`TASK.yaml.subject.n_seeds` is `null` / not
applicable here — this experiment is deterministic synthesis + a
Monte-Carlo TVLA estimate, not a trained model with run-to-run variance);
the TVLA RNG seed is fixed (`12345`) for reproducibility of the reported
numbers, and `functional_check_*` uses its own fixed seeds (`42`/`43`).

## Dataset

None (`TASK.yaml.datasets: []`). The two benchmarks are RTL source files
copied verbatim (with one documented portability patch, see
`rtl_src/aes_sbox_dom_reg.v` and the note below) from the upstream
`nilotpolas/MaskedHLS` GitHub repository, which is pinned as the
`under_test` subject in `TASK.yaml.subject.models`.

## RTL portability patch (documented, not a design change)

`4. SecureHLS-main/test/AES/DOM/output.v` in the upstream repo declares its
own clock port as `input [7:0] clk;` (an 8-bit bus) even though `clk` is
never used as data anywhere else in the module and every testbench in the
repo drives it as a scalar reg. Real Verilog semantics require an
edge-sensitivity signal (`posedge`/`negedge`) to be exactly 1 bit wide, and
yosys enforces this strictly (rejecting the file with a frontend error);
some commercial simulators are more lenient and silently accept it. The
copy at `rtl_src/aes_sbox_dom_reg.v` changes that one line to
`input clk;` — no other line was touched, and `clk`'s only other use in the
file is the `always @(posedge clk)` sensitivity list, so this cannot change
behavior.
